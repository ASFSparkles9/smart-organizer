# Где находится "AI" и как он работает

## ⚠️ Важно: Сейчас НЕТ настоящей нейросети!

Текущая реализация использует **алгоритмический анализ** с правилами и регулярными выражениями, а не машинное обучение.

## 📍 Где находится код "AI"

### Основной файл: `src/lib/ai/localAI.ts`

Этот файл содержит класс `LocalAI` с методами анализа:

1. **`analyzeNote()`** - анализ заметок (строки 127-375)
   - Использует регулярные выражения для поиска паттернов
   - Ищет имена, адреса, телефоны по шаблонам
   - Определяет категорию по ключевым словам
   - НЕ использует нейросеть

2. **`extractTasksFromNote()`** - извлечение задач (строки 104-125)
   - Ищет паттерны: "нужно", "надо", "todo"
   - Регулярные выражения
   - НЕ использует нейросеть

3. **`sortNotes()`** - сортировка заметок (строки 377-404)
   - Сортирует по релевантности (вычисленной алгоритмически)
   - НЕ использует нейросеть

## 🔍 Что на самом деле происходит

### Анализ заметки (`analyzeNote`):

```typescript
// Это НЕ нейросеть, а простые правила:
- Ищет имена по паттерну: /[А-ЯЁ][а-яё]+\s+[А-ЯЁ][а-яё]+/
- Ищет адреса по словам: "улица", "балтийская" + числа
- Ищет телефоны по шаблону: /\d{3}-\d{3}-\d{2}-\d{2}/
- Определяет категорию по ключевым словам из словаря
- Извлекает теги, исключая стоп-слова
```

### Пример работы:

**Вход:** "учебная часть оки 230 231 татьяна александровна"

**Что делает код:**
1. Находит "Татьяна Александровна" (паттерн имени)
2. Находит "230", "231" (числа)
3. Видит "учебная" → категория = "study"
4. Формирует главную мысль из найденных сущностей

**Это НЕ нейросеть** - это просто правила и шаблоны!

## 🚀 Как добавить настоящую нейросеть

### Вариант 1: WebLLM (локально в браузере)

```typescript
// Установка
npm install @mlc-ai/web-llm

// В src/lib/ai/localAI.ts
import { LLM } from '@mlc-ai/web-llm';

class LocalAI {
  private llm: LLM | null = null;

  async init() {
    this.llm = new LLM({
      model: 'Llama-2-7b-chat-hf-q4f32_1',
    });
    await this.llm.load(); // Загрузка ~4GB модели
  }

  async analyzeNote(note: Note) {
    if (!this.llm) await this.init();
    
    const prompt = `Проанализируй заметку и определи:
    1. Главную мысль (1-2 предложения)
    2. Категорию (работа/учеба/здоровье/личное)
    3. Ключевые теги (3-5 слов)
    
    Заметка: ${note.content}`;
    
    const response = await this.llm.generate(prompt);
    // Парсинг ответа...
  }
}
```

**Плюсы:**
- Работает полностью локально
- Не отправляет данные на сервер
- Бесплатно

**Минусы:**
- Первая загрузка: 10-30 секунд (~4GB)
- Использование памяти: 2-4GB
- Медленнее, чем API

### Вариант 2: Transformers.js (легче, но менее мощно)

```typescript

npm install @xenova/transformers

import { pipeline } from '@xenova/transformers';

class LocalAI {
  private classifier: any = null;

  async init() {
    this.classifier = await pipeline(
      'text-classification',
      'Xenova/distilbert-base-uncased' // ~250MB
    );
  }

  async analyzeNote(note: Note) {
    if (!this.classifier) await this.init();
    
    // Классификация категории
    const categoryResult = await this.classifier(note.content);
    // Извлечение сущностей и т.д.
  }
}
```

**Плюсы:**
- Меньше размер (~250MB)
- Быстрее загружается

**Минусы:**
- Менее мощная модель
- Нужны отдельные модели для разных задач
